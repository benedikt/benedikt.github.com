flinc

From late 2010 to early 2013 I worked full-time at flinc. I helped bringing the vision of a real-time ride searching system to life and to more than 100,000 users. During this time I helped both composing and implementing the system. I worked on components such as complex database queries, integrating third party services, background processing, as well as front-end development, including both user interface logic and design. Additionally I helped keeping the system fast and scalable by removing performance bottlenecks and optimising several parts of the application.

Technologies involved:
  • PostgreSQL
  • PostGIS
  • Redis
  • Memcached
  • Ruby on Rails
  • RSpec
  • Resque
  • HAML
  • SASS (with Compass)

InstantNavi

During my time at flinc, I also worked on flinc's other product, InstantNavi. It provides in-browser satellite navigation using HTML5 technologies. It is backed by a management application built with Ruby on Rails and MongoDB, that allows creating and managing of differently styled satellite navigation setups. I helped implementing the backing application, as well as improving the JavaScript based satellite navigation framework.

Technologies involved:
  • Ruby on Rails
  • JavaScript
  • MongoDB
  • OpenStreetMaps
  • RSpec
  • HAML
  • SASS (with Compass)
